# Налаштування автозапуску проекту "Облік ОЗ" при завантаженні Windows

## Що робить скрипт

Файл `start-docker-project.bat` автоматично:
1. Чекає поки Docker Desktop повністю запуститься
2. Запускає всі контейнери проекту (`docker compose up -d`)
3. Закриває вікно через 10 секунд

Після запуску проект доступний:
- **Frontend:** http://localhost
- **Backend:** http://localhost:8000

---

## Передумови

- Docker Desktop встановлено і налаштовано на автозапуск при завантаженні Windows
  - Відкрити Docker Desktop → Settings → General → **Start Docker Desktop when you sign in** (увімкнено)

---

## Як додати в автозавантаження

### Спосіб 1: Через папку автозавантаження (найпростіший)

1. Натиснути `Win + R`
2. Ввести `shell:startup` і натиснути Enter — відкриється папка автозавантаження
3. Скопіювати файл `C:\Buh\start-docker-project.bat` в цю папку

> Або створити ярлик: ПКМ по файлу → Створити ярлик → перемістити ярлик в папку `shell:startup`

### Спосіб 2: Через Планувальник завдань (більше контролю)

1. Натиснути `Win + R`, ввести `taskschd.msc`, натиснути Enter
2. В правій панелі натиснути **"Створити просте завдання..."**
3. Назва: `Buh Docker Autostart`, натиснути Далі
4. Тригер: **"При вході в систему"**, натиснути Далі
5. Дія: **"Запустити програму"**, натиснути Далі
6. Програма: `C:\Buh\start-docker-project.bat`, натиснути Далі
7. Натиснути **Готово**

Додатково (необов'язково):
- Відкрити створене завдання → Властивості
- Вкладка "Загальні" → поставити **"Виконувати з найвищими правами"**
- Вкладка "Параметри" → зняти **"Зупиняти завдання, яке виконується довше"**

---

## Як видалити з автозавантаження

### Якщо використовували Спосіб 1:
1. `Win + R` → `shell:startup` → Enter
2. Видалити файл `start-docker-project.bat` (або ярлик)

### Якщо використовували Спосіб 2:
1. `Win + R` → `taskschd.msc` → Enter
2. Знайти завдання `Buh Docker Autostart`
3. ПКМ → Видалити
