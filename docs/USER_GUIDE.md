## 🔐 Вхід у систему

### Як увійти

1. Відкрийте браузер та перейдіть за адресою системи (зазвичай `http://localhost:5173`)
2. На сторінці входу введіть:
   - **Ім'я користувача** — логін, який вам надав адміністратор
   - **Пароль** — ваш пароль
3. Натисніть кнопку **«Увійти»**

### 💡 Примітки

- Сесія діє **8 годин**, після чого система автоматично продовжить її (якщо ви активні)
- Якщо сесія закінчилась — система перенаправить на сторінку входу
- Для виходу натисніть на ваше ім'я у верхньому правому куті → **«Вийти»**

## 📋 Реєстр основних засобів

**Навігація:** Бокове меню → 📋 **Основні засоби** → **Реєстр ОЗ**

Головна таблиця з переліком усіх основних засобів організації.

### Можливості

| Дія | Як виконати |
|-----|------------|
| 🔍 **Пошук** | Введіть текст у поле пошуку (шукає за назвою та інвентарним номером) |
| 🔽 **Фільтр за статусом** | Оберіть: Активний / Списаний / Законсервований |
| ➕ **Додати ОЗ** | Натисніть кнопку **«+ Додати ОЗ»** у верхньому правому куті |
| ✏️ **Редагувати** | Натисніть на рядок таблиці для переходу до картки ОЗ |
| 🗑️ **Видалити** | Кнопка видалення в рядку таблиці (тільки якщо немає операцій) |
| 📄 **Експорт** | Кнопка завантаження PDF або Excel |

### Поля таблиці

| Колонка | Опис |
|---------|------|
| Інвентарний номер | Унікальний номер ОЗ |
| Назва | Найменування об'єкта |
| Група ОЗ | Код та назва групи згідно ПКУ |
| Первісна вартість | Вартість придбання, грн |
| Залишкова вартість | Поточна балансова вартість, грн |
| Знос | Накопичена амортизація, грн |
| Статус | Активний / Списаний / Законсервований |
| МВО | Матеріально відповідальна особа |
| Місцезнаходження | Де розташований об'єкт |

### Створення нового ОЗ

1. Натисніть **«+ Додати ОЗ»**
2. Заповніть обов'язкові поля:
   - **Інвентарний номер** — унікальний ідентифікатор
   - **Назва** — найменування об'єкта
   - **Група ОЗ** — оберіть зі списку (1-16)
   - **Первісна вартість** — вартість у гривнях
   - **Дата введення в експлуатацію**
   - **Метод амортизації** — один із 5 методів
   - **Строк корисного використання** — у місяцях
3. Заповніть додаткові поля (за потреби):
   - Ліквідаційна вартість, МВО, Місцезнаходження, Опис
4. Натисніть **«Зберегти»**

## 📥 Прихід (оприбуткування)

**Навігація:** Бокове меню → 📋 **Операції** → **Прихід**

Реєстрація надходження основних засобів.

### Типи надходження

| Тип | Опис |
|-----|------|
| 🛒 Придбання | Купівля за грошові кошти |
| 🎁 Безоплатне отримання | Отримання від інших організацій |
| 💼 Внесок до статутного капіталу | Від засновників |
| 🔄 Обмін | В результаті бартерної операції |
| 🔨 Самостійне виготовлення | Створення власними силами |
| 📎 Інше | Інші джерела надходження |

### Як оформити прихід

1. Перейдіть на сторінку **Прихід**
2. Натисніть **«+ Новий прихід»**
3. Заповніть поля:
   - **Основний засіб** — оберіть зі списку (або створіть новий)
   - **Тип надходження** — оберіть зі списку вище
   - **Номер документа** — номер первинного документа
   - **Дата документа** — дата первинного документа
   - **Постачальник** — назва постачальника (за наявності)
   - **Сума** — вартість за документом, грн
   - **Примітки** — додаткова інформація
4. Натисніть **«Зберегти»**

### 📄 Документи

- **PDF (ОЗ-1)** — Акт приймання-передачі основних засобів
- **Excel (ОЗ-1)** — те саме у форматі Excel
- Кнопки експорту доступні в рядку кожного запису

### 💡 Що відбувається автоматично

- ✅ Створюється бухгалтерська проводка (Дт рахунок обліку — Кт джерело)
- ✅ Записується в журнал аудиту
- ✅ Надсилається сповіщення

## 📉 Амортизація

**Навігація:** Бокове меню → 📉 **Амортизація**

Нарахування щомісячної амортизації основних засобів.

### Як нарахувати амортизацію

1. Перейдіть на сторінку **Амортизація**
2. Оберіть **рік** та **місяць** для нарахування
3. Натисніть кнопку **«Нарахувати»**
4. Система автоматично обчислить амортизацію для всіх активних ОЗ
5. Перегляньте результати в таблиці

### 5 методів амортизації

| # | Метод | Як працює |
|---|-------|-----------|
| 1️⃣ | **Прямолінійний** | Однакова сума кожного місяця. Формула: (Первісна вартість - Ліквідаційна вартість) / Строк використання |
| 2️⃣ | **Зменшення залишкової вартості** | Фіксований відсоток від залишку. Більше на початку, менше — наприкінці |
| 3️⃣ | **Прискореного зменшення** | Подвійна норма прямолінійного методу від залишкової вартості |
| 4️⃣ | **Кумулятивний** | Коефіцієнт = залишок років / сума чисел років. Найбільший знос на початку |
| 5️⃣ | **Виробничий** | Пропорційно фактичному обсягу продукції. Потрібно вказати обсяг за місяць |

### 📄 Документи

- **PDF / Excel** — Відомість нарахованої амортизації за обраний період

### 💡 Поради

- Нарахування потрібно робити **щомісяця** (як правило, в кінці місяця)
- Амортизація не нараховується на повністю замортизовані ОЗ
- Амортизація не нараховується на списані та законсервовані ОЗ
- Система попередить, якщо амортизація за цей місяць вже нарахована

## 📦 Інвентаризація

**Навігація:** Бокове меню → 📦 **Інвентаризація**

Проведення інвентаризації необоротних активів згідно з Наказом Мінфін №572.

### Етапи інвентаризації

#### Крок 1️⃣ — Створення

1. Натисніть **«+ Нова інвентаризація»**
2. Заповніть:
   - **Номер** — номер інвентаризаційного документа
   - **Дата** — дата проведення
   - **Номер розпорядчого документа** — наказ про проведення
   - **Місцезнаходження** — де проводиться (за потреби)
   - **Голова комісії** — оберіть зі списку користувачів
   - **Члени комісії** — оберіть одного або кількох
3. Натисніть **«Зберегти»**

#### Крок 2️⃣ — Заповнення

1. Відкрийте створену інвентаризацію
2. Натисніть **«Заповнити активами»** — система додасть усі активні ОЗ автоматично
3. Або додайте ОЗ вручну

#### Крок 3️⃣ — Проведення

Для кожного ОЗ в списку:
1. Позначте **«Знайдено»** ✅ або залиште порожнім ❌
2. Вкажіть **стан**: Добрий / Потребує ремонту / Непридатний
3. За потреби додайте **примітки**

#### Крок 4️⃣ — Завершення

1. Переконайтесь, що всі ОЗ перевірені
2. Натисніть **«Завершити інвентаризацію»**
3. Система автоматично обчислить розбіжності

### 📄 Документи

- **PDF** — Інвентаризаційний опис необоротних активів (16-колонкова форма згідно Наказу №572)
- **Excel** — те саме у форматі Excel

## 💰 Переоцінки

**Навігація:** Бокове меню → 📋 **Операції** → **Переоцінки**

Переоцінка основних засобів згідно НП(С)БО 7, п.16-21.

### Як провести переоцінку

1. Натисніть **«+ Нова переоцінка»**

### 💡 Що відбувається автоматично

- ✅ Визначається тип: **дооцінка** (↑) або **уцінка** (↓)
- ✅ Обчислюється коефіцієнт переоцінки: `Справедлива вартість / Залишкова вартість`
- ✅ Перераховується первісна вартість та знос пропорційно
- ✅ Оновлюються дані ОЗ
- ✅ Створюються бухгалтерські проводки

## ⏰ Автоматичні задачі (Celery)

Система має фонові задачі, які виконуються автоматично за розкладом. Для їх роботи потрібні **Redis**, **Celery Worker** та **Celery Beat**.

### Запуск

- **Windows:** двічі клікніть `start-celery.bat`
- **Docker:** запускаються автоматично (контейнери `buh_celery` та `buh_celery_beat`)

### Розклад задач

| Задача | Коли виконується | Що робить |
|--------|-----------------|-----------|
| 📧 **Нагадування про амортизацію** | 1 число кожного місяця, 09:00 | Надсилає сповіщення бухгалтерам про необхідність нарахувати амортизацію |
| ⚠️ **Перевірка зношеності ОЗ** | Щопонеділка, 08:00 | Знаходить ОЗ зі зносом >90% та створює сповіщення |
| 🔴 **Перевірка повної амортизації** | Щопонеділка, 08:30 | Знаходить повністю замортизовані ОЗ та створює сповіщення |
| ☁️ **Автоматичне резервне копіювання** | За розкладом (налаштовується) | Створює дамп бази та завантажує на Google Drive |

### 💡 Важливо

- Без запущеного Celery сповіщення про амортизацію та знос **не будуть надсилатися**
- Автоматичне резервне копіювання на Google Drive **не працюватиме** без Celery
- Всі інші функції системи (CRUD, документи, звіти) працюють **без Celery**
