🖥️ Інструкція з використання BAT-скриптів

Облік ОЗ — Повний набір скриптів для встановлення, налаштування, запуску та зупинки системи

🗺️ Загальна схема

Скрипти виконуються в певному порядку. Ось повний шлях від чистої системи до працюючого додатку:

🖥️ ЧИСТА СИСТЕМА WINDOWS

• Python 3.12   • Node.js LTS   • PostgreSQL 16

⚙️ setup.bat
Налаштовує:
• .env конфігурація   • Python venv + pip
• База даних + міграції   • Початкові дані   • npm install

🚀 start.bat
Запускає:
• Django backend   • React frontend   • Відкриває браузер

🔧 install-prerequisites.bat

📝 Опис

Автоматично встановлює всі необхідні програми для роботи системи через Windows Package Manager (winget). Цей скрипт потрібно запустити лише один раз на новому комп'ютері.

📦 Що встановлює

Програма Версія Навіщо потрібна
🐍 Python 3.12 Мова програмування для backend (Django)
🟩 Node.js LTS (22.x) Середовище для frontend (React + Vite)
🐘 PostgreSQL 16 База даних

🚀 Як запустити

  1. Натисніть правою кнопкою миші на файл install-prerequisites.bat
  2. Виберіть «Запустити від імені адміністратора»
  3. Підтвердіть запит UAC (Windows запитає дозвіл)
💡 Підказка: Якщо запустити без прав адміністратора — скрипт автоматично перезапуститься з UAC-промптом.

  • Якщо потрібно встановити все: 10-15 хвилин (залежить від швидкості інтернету)

🔑 Крок 4 — Налаштування .env (детально)

На кроці 4 автоматично відкриється Блокнот з файлом backend\.env. Вам потрібно встановити пароль PostgreSQL:

## Налаштування системи обліку основних засобів

# Django
DJANGO_SECRET_KEY=django-insecure-auto-generated-123456789
DEBUG=True

# PostgreSQL
POSTGRES_DB=buh_assets
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password_here    ← ⚠️ ЗМІНІТЬ НА СВІЙ ПАРОЛЬ!
POSTGRES_HOST=localhost
POSTGRES_PORT=5432

# CORS
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
⚠️ Важливо: Значення POSTGRES_PASSWORD повинно збігатися з паролем, який ви задали при встановленні PostgreSQL. Якщо ви використовували install-prerequisites.bat, це пароль з установника PostgreSQL.

🔑 Крок 7 — Створення суперкористувача (детально)

Скрипт запитає дані для адміністратора системи:

Поле Що вводити Приклад
👤 Username Логін адміністратора admin
📧 Email Електронна пошта admin@company.com
🔒 Password Пароль (не менше 8 символів) MySecurePass123
🔒 Password (again) Повторіть пароль MySecurePass123
💡 Підказка: Ці дані використовуються для входу в систему за адресою http://localhost:5173 та в Django Admin за адресою http://localhost:8000/admin/

⏱️ Тривалість

  • Типовий час: 3-7 хвилин (залежить від швидкості інтернету та диска)
  • Найдовший крок — встановлення Python-залежностей (pip install)

⚠️ Можливі помилки

Помилка Причина Рішення
Python не знайдено Python не встановлений або не в PATH Запустіть install-prerequisites.bat
Node.js не знайдено Node.js не встановлений Запустіть install-prerequisites.bat
Не вдалося створити вiртуальне середовище Пошкоджений Python Перевстановіть Python
Не вдалося встановити Python-залежностi Проблеми з мережею або pip cd backend & venv\Scripts\activate & pip install -r requirements.txt
Мiграцiї не вдалися PostgreSQL не запущений або невірний пароль Перевірте службу PostgreSQL та backend\.env
npm install не вдався Node.js не в PATH або проблеми з мережею cd frontend & npm install

🔄 Гаряче перезавантаження (HMR)

Vite автоматично оновлює сторінку в браузері при зміні файлів. Не потрібно перезапускати сервер після кожної зміни коду!

⚠️ Важливо: Frontend потребує працюючий backend для роботи з даними. Якщо backend не запущений, ви побачите помилки при спробі завантажити дані.

🔧 Методи зупинки

Скрипт використовує два методи для надійної зупинки:

  1. 🗔️ За назвою вікна — закриває всі вікна з назвою Oblik OZ*
  2. 🔌 За портом — знаходить і завершує процеси, які слухають порти 8000 та 5173
💡 Навіть якщо ви закрили вікна вручну, але процеси «зависли» — stop.bat їх знайде і завершить.

❓ Часті питання (FAQ)

🔹 Який порядок запуску на новому комп'ютері? 
1️⃣  install-prerequisites.bat     (один раз — встановлює програми)
2️⃣  setup.bat                     (один раз — налаштовує проєкт)
3️⃣  start.bat                     (щоразу — запускає систему)
🔹 Я вже маю Python/Node.js/PostgreSQL, що запускати?

Пропустіть install-prerequisites.bat і одразу запустіть:

1️⃣  setup.bat     (налаштування проєкту)
2️⃣  start.bat     (запуск системи)
🔹 Як запустити систему наступного разу?

Просто запустіть start.bat — він зробить все необхідне. Повторно запускати setup.bat чи install-prerequisites.bat не потрібно.

🔹 Чи можна запускати Backend та Frontend окремо?

Так! Це зручно при розробці:

:: В одному терміналі:
start-backend.bat

:: В іншому терміналі:
start-frontend.bat
🔹 Як змінити порти?
Сервіс Файл Де змінити

rmdir /s /q backend\venv

🔹 Чи можна поставити систему на іншому ПК через локальну мережу?

Так! Після запуску start.bat система доступна з інших комп'ютерів у тій самій мережі:

  • Дізнайтесь вашу IP-адресу: ipconfig
  • Відкрийте на іншому ПК: http://<ваш-IP>:5173
⚠️ Переконайтесь, що Windows Firewall дозволяє вхідні з'єднання на порти 5173 та 8000.

🔥 Вирішення проблем

🐍 Python

❌ «Python не знайдено»

Причина: Python не встановлений або не доданий у PATH.

Рішення:

  1. Запустіть install-prerequisites.bat
  2. Або встановіть вручну з https://www.python.org/downloads/
  3. ⚠️ При встановленні обов'язково поставте галочку «Add Python to PATH»
  4. Після встановлення перезапустіть командний рядок

Перевірка:

python --version
:: Має вивести: Python 3.12.x
❌ «Не вдалося створити вiртуальне середовище»

Причина: Пошкоджена інсталяція Python або відсутній модуль venv.

Рішення:

  1. Видаліть Python через «Програми та компоненти»
  2. Перевстановіть Python з https://www.python.org/downloads/
  3. При встановленні оберіть «Customize installation» та переконайтесь, що ✅ pip та ✅ venv відмічені
  4. Запустіть setup.bat повторно
❌ «Не вдалося встановити Python-залежностi»

Причина: Проблеми з мережею, старий pip, або несумісні пакети.

Рішення:

cd backend
venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt --verbose

Прочитайте повідомлення про помилку — зазвичай воно вказує на конкретний пакет.

🟩 Node.js

❌ «Node.js не знайдено»

Причина: Node.js не встановлений або не в PATH.

Рішення:

  1. Запустіть install-prerequisites.bat
  2. Або встановіть вручну з https://nodejs.org/ (оберіть LTS-версію)
  3. Перезапустіть командний рядок

Перевірка:

node --version
npm --version
❌ «npm install не вдався»

Причина: Проблеми з мережею, кеш npm або несумісні залежності.

Рішення:

cd frontend
npm cache clean --force
npm install

Якщо не допомогло:

rmdir /s /q node_modules
del package-lock.json
npm install

🐘 PostgreSQL

❌ «psql не знайдено в PATH»

Причина: PostgreSQL встановлений, але шлях до bin не в PATH.

Рішення:

  1. Знайдіть папку PostgreSQL (зазвичай C:\Program Files\PostgreSQL\16\bin)
  2. Додайте її у змінну PATH:
    • 🗔️ Win+R → sysdm.cpl → Додатково → Змінні середовища
    • У Системних змінних знайдіть Path → Редагувати
    • Додайте: C:\Program Files\PostgreSQL\16\bin
    • Натисніть OK → OK
  3. Перезапустіть командний рядок

Перевірка:

psql --version
:: Має вивести: psql (PostgreSQL) 16.x
❌ «Мiграцiї не вдалися»

Причина: PostgreSQL не запущений, невірний пароль, або база даних не існує.

Рішення:

  1. Перевірте чи PostgreSQL запущений:
    • 🗔️ Win+R → services.msc
    • Знайдіть postgresql-x64-16
    • Стан повинен бути: «Виконується» / «Running»
    • Якщо зупинений → натисніть правою кнопкою → «Запустити»
  2. Перевірте пароль у backend\.env: 
    POSTGRES_PASSWORD=ваш_реальний_пароль
  3. Перевірте чи база існує: 
    psql -U postgres -c "\l"
:: Шукайте buh_assets у списку
  4. Створіть базу вручну: 
    create-db.bat
:: або
psql -U postgres -c "CREATE DATABASE buh_assets ENCODING 'UTF8';"
❌ PostgreSQL запитує пароль, а я його не пам'ятаю

Рішення:

  1. Відкрийте файл C:\Program Files\PostgreSQL\16\data\pg_hba.conf
  2. Знайдіть рядок: 
    host  all  all  127.0.0.1/32  scram-sha-256
  3. Змініть scram-sha-256 на trust
  4. Перезапустіть службу PostgreSQL (services.msc)
  5. Тепер можна змінити пароль: 
    psql -U postgres -c "ALTER USER postgres PASSWORD 'новий_пароль';"
  6. Поверніть trust назад на scram-sha-256
  7. Перезапустіть PostgreSQL
  8. Оновіть пароль у backend\.env

🗔️ Windows / Загальні

❌ «winget не знайдено»

Причина: Windows Package Manager (App Installer) не встановлений або застаріла версія Windows.

Рішення:

  1. Відкрийте Microsoft Store
  2. Знайдіть «App Installer» від Microsoft
  3. Натисніть «Оновити» або «Встановити»
  4. Перезапустіть install-prerequisites.bat
ℹ️ winget доступний на Windows 10 (1709+) та Windows 11.
❌ Програми встановились, але «не знайдено»

Причина: PATH не оновився в поточному вікні.

Рішення:

  1. Закрийте всі вікна командного рядка
  2. Відкрийте нове вікно
  3. Запустіть скрипт повторно

Або перезавантажте комп'ютер.

❌ Скрипт показує кракозябри замість українських літер

Причина: Неправильна кодова сторінка терміналу.

Рішення:

Усі скрипти вже містять chcp 65001 для переключення на UTF-8. Якщо проблема залишається:

  1. 🗔️ Win+R → intl.cpl
  2. Вкладка «Адміністрування» → «Змінити мову системи»
  3. Поставте ✅ «Beta: Use Unicode UTF-8 for worldwide language support»
  4. Перезавантажте комп'ютер
❌ Порт 8000 або 5173 вже зайнятий

Причина: Інший процес вже використовує порт.

Рішення:

:: 1. Зупиніть всі сервіси
stop.bat

:: 2. Перевірте що займає порт
netstat -aon | findstr :8000
netstat -aon | findstr :5173

:: 3. Якщо процес ще живий, завершіть його за PID
taskkill /f /pid <PID>
